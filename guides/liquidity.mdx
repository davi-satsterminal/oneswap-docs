## Add liquidity

```typescript
const lp = await client.liquidity.create({
  poolId: 'Amulet-USDCx',
  amountA: '100',
  amountB: '150',
  walletAddress: 'alice::12205a8c...'
})

console.log(lp.depositAddress) // pool LP party
console.log(lp.expiresAt)
```

Both deposits must be sent from the same party specified in `walletAddress`.

## Direct-party LP flow

1. Create the LP intent with the user's Canton party
2. OneSwap returns the pool LP party as `depositAddress`
3. Send token A and token B directly from that same party to `depositAddress`
4. OneSwap matches the two deposits and credits the LP position in the database

## Track LP status

```typescript
lp.on('partial', () => console.log('One token received'))
lp.on('completed', () => console.log('LP position credited'))

const result = await lp.wait()
```

Or poll manually:

```typescript
const status = await client.liquidity.getStatus(lp.intentId)
```

## Status lifecycle

```
pending → partial → processing → completed
        → expired
```

## View positions

```typescript
const positions = await client.positions.list('alice::12205a8c...')

for (const pos of positions.positions) {
  console.log(`${pos.poolName}: ${pos.sharePercent}% share`)
  console.log(`  ${pos.tokenA}: ${pos.valueTokenA}`)
  console.log(`  ${pos.tokenB}: ${pos.valueTokenB}`)
}
```

## Check earnings

```typescript
const earnings = await client.positions.getEarnings('alice::12205a8c...', 'Amulet-USDCx')

console.log(`APR: ${earnings.apr}%`)
console.log(`Projected daily earnings: ${earnings.estimatedDailyEarnings}`)
console.log(`LP share: ${earnings.lpSharePercent}%`)
```

Use `client.liquidity.list()` to inspect recent LP intents created through your API key.
