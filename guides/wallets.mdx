## What `client.wallets` does now

The SDK uses the user's real Canton party directly.

`client.wallets` is now a lightweight registry for Canton party IDs tied to your developer account. It lets you:

- register a user party once
- list the parties you use through the SDK
- query live balances for those parties
- scope position and tracking queries to known parties

## Register a party

```typescript
const wallet = await client.wallets.create({ walletAddress: 'alice::12205a8c...' })

console.log(wallet.walletId)
console.log(wallet.walletAddress) // user's Canton party ID
console.log(wallet.address)       // same party ID
console.log(wallet.createdAt)
```

## Get a registered party

```typescript
const wallet = await client.wallets.get('alice::12205a8c...')
console.log(wallet.address)
```

## List registered parties

```typescript
const result = await client.wallets.list()
for (const w of result.wallets) {
  console.log(w.walletAddress)
}
```

## Check balances

```typescript
const balances = await client.wallets.getBalances('alice::12205a8c...')
for (const b of balances.balances) {
  console.log(`${b.token}: ${b.amount}`)
}
```

These are live on-chain balances for that party, aggregated by token.

## Use a registered party with swaps

```typescript
const swap = await client.swaps.create({
  fromToken: 'Amulet',
  toToken: 'USDCx',
  amount: '100',
  walletAddress: 'alice::12205a8c...'
})
```

The `walletAddress` is the actual sender party. The returned `depositAddress` will be the pool swap party, not the user's own party.
