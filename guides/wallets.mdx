## Wallet methods are no longer part of the SDK

The public SDK no longer exposes `client.wallets.*`.

- Pass the user's Canton `party ID` directly into `client.swaps.create({ walletAddress })`.
- Use your own wallet provider or app state to show balances.
- Use the main OneSwap website for liquidity management.

## Example swap flow

```typescript
const swap = await client.swaps.create({
  fromToken: 'Amulet',
  toToken: 'USDCx',
  amount: '100',
  walletAddress: 'alice::12205a8c...'
})
```

The `walletAddress` is the actual sender party. The returned `depositAddress` will be the pool swap party, not the user's own party.
