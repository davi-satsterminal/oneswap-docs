title: Wallets
description: Create and manage custodial wallets for your users.
---

## Why custodial wallets

Your users don't need their own Canton party or wallet setup. You create a wallet for them, and it's managed through your API key.

## Create a wallet

```typescript
const wallet = await client.wallets.create({ walletAddress: 'alice::12205a8c...' })

// wallet.walletId      — internal UUID
// wallet.walletAddress — "alice::12205a8c..."
// wallet.address       — Canton party ID (use this for deposits/transfers)
// wallet.createdAt     — timestamp
```

The `walletAddress` is your user's Canton party ID — from PartyLayer wallet connect or any Canton wallet.

## Get a wallet

```typescript
const wallet = await client.wallets.get('alice::12205a8c...')
console.log(wallet.address) // Canton party ID
```

## List all wallets

```typescript
const result = await client.wallets.list()
for (const w of result.wallets) {
  console.log(`${w.walletAddress}: ${w.address}`)
}
```

## Check balances

```typescript
const balances = await client.wallets.getBalances('alice::12205a8c...')
for (const b of balances.balances) {
  console.log(`${b.token}: ${b.amount}`)
}
```

## Use wallets with swaps

Pass `walletAddress` when creating swaps or LP intents to use the custodial wallet:

```typescript
const swap = await client.swaps.create({
  fromToken: 'Amulet',
  toToken: 'USDCx',
  amount: '100',
  walletAddress: 'alice::12205a8c...' // deposit address will be this wallet
})
```

## Use a custom party instead

If your user has their own Canton party (e.g. from a wallet like Loop), use `senderParty` instead of `walletAddress`:

```typescript
const swap = await client.swaps.create({
  fromToken: 'Amulet',
  toToken: 'USDCx',
  amount: '100',
  senderParty: 'user-party-id::namespace'
})
```
