---
title: Fee Earnings
description: Track SDK developer fee earnings and direct-payout collections.
---

OneSwap shares a portion of platform swap fees with SDK developers. Once your account is enabled for fee sharing, fees accumulate automatically on every swap made through your API keys.

## How it works

1. A user executes a swap through your integration
2. The platform takes its standard fee
3. Your configured share is credited to your developer account in the database
4. When you collect earnings, payout is sent directly to the destination party you specify in the developer portal or admin tooling

## Check your earnings

```typescript
const earnings = await client.fees.getEarnings()

console.log(earnings.feeSharePct)
console.log(earnings.payoutMode)     // 'direct'
console.log(earnings.accumulated)
console.log(earnings.totalCollected)
```

`feeWallet` is `null` in direct mode. There is no permanent custodial fee wallet anymore.

## View fee event history

```typescript
const history = await client.fees.getEarningsHistory({
  page: 1,
  limit: 20,
  token: 'CC',
})
```

## View collection history

```typescript
const collections = await client.fees.getCollections({ page: 1, limit: 20 })

for (const col of collections.collections) {
  console.log(`${col.amount} ${col.tokenName} sent to ${col.destinationParty ?? 'destination'} at ${col.collectedAt}`)
}
```

## Developer portal

Use the developer portal to review accumulated fees and initiate collections. Each collection asks for a destination Canton party and transfers directly there.
